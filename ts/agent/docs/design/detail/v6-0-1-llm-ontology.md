该系统的设计与实现围绕着将 AI 从简单的“图搜索工具”进化为“工业级决策支持系统”这一核心目标，经历了从 V5 到 V7 的深度演进。其核心框架可以总结为：**本体（Ontology）定义规范、图（Graph）提供事实、Agent（Agent）执行受控推理**。

以下是关于这三个维度的详细设计与实现总结：

### 1. 本体设计 (Ontology: G = {E, R, T, C})
本体不再仅仅是静态的 Schema，而是被视为**“本体即代码 (Ontology-as-code)”**的工程化资产。
*   **C（准则/规则）的一等公民化**：在 V5 中，规则（Constraints）从隐含的代码逻辑变为可查询、可引用的运行时对象，包含硬约束、软准则和推理规则。
*   **Rule DAG（规则有向无环图）**：V6 引入拓扑排序，确保执行顺序为：推理规则（产生推导事实）→ 硬约束（执行一票否决 Veto）→ 软准则（加权打分）。
*   **因果建模 (CausalGraph)**：V6.5 区分了结构化关系（R）与因果机制，通过因果图支持后向归因（RCA）和“But-for”反事实测试。

### 2. 图数据与事实管理 (Graph & Facts)
系统放弃了全量数据注入，转向**渐进式披露**和**结构化事实绑定**。
*   **渐进式实体披露 (Progressive Disclosure)**：System Prompt 仅提供入口实体和 Schema，Agent 必须通过 `inspect_node` 和 `query_neighbors` 按需探索图，以适配大规模数据和隐私权限场景。
*   **FactStore 与 FactBinding**：
    *   **结构化存储**：弃用 V5 的扁平 KV（如 `workload_alice`），改用 `(entityId, property)` 命名空间绑定。
    *   **五类来源 (Source)**：事实标注来源类型，包括图属性（graph_property）、方法结果（method_result）、聚合（aggregation）、用户输入（user_input）及推导（derived）。
    *   **时效性与置信度**：每个事实带有 `validUntil`（有效期）和由系统计算（而非模型编造）的 `confidence`。
*   **权限与隐私 (PolicyContext)**：在工具层通过装饰器强制执行权限过滤，对敏感字段（PII）进行遮蔽（Masking），并禁止将脱敏数据记入证据链。

### 3. Agent 架构与推理逻辑 (Agent Loop)
Agent 的角色从“会用图的工具人”升级为**“受控的决策助手”**，采用 **Planner-Executor-Critic** 三段式架构。
*   **三段式流程**：
    *   **Planner**：轻量级计划，根据意图预判需调查的子图，避免盲目搜索。
    *   **Executor**：受控执行。职责限定为探索图、绑定事实和提议候选，**严禁自行打分**。
    *   **Deterministic Critic**：确定性评价器。不使用 LLM，而是基于 **MCDA（多准则决策分析）**和 Rule DAG 运行，确保评分可重放、可校准。
*   **判决双轨制 (Dual-track Verdict)**：
    *   同时输出**系统判决**（确定性规则结果）与**模型判决**（语义感性判断）。
    *   **协调器 (Reconciler)**：显式暴露两者的冲突并分析原因，由用户做最终裁决，防止模型静默覆盖系统逻辑。
*   **工具层进化**：
    *   `call_method` 增加**前置断言 (Precondition)**，防止模型在缺乏事实的情况下用零值参数进行盲调。
    *   提供 `simulate` 工具支持**反事实重放**，允许局部替换事实并观察结论变化。

### 4. 核心实现对比总结 (V5 vs. V6/V7)

| 维度 | V5 设计 | V6/V7 实现 |
| :--- | :--- | :--- |
| **前端入口** | 调用方写死入口实体 | **自然语言 → 意图识别 + 实体链接** |
| **事实存储** | 扁平 KV (`Record<string, any>`) | **FactStore (entityId + property 绑定)** |
| **评分机制** | 触发规则数加权 (简单比例) | **MCDA (权重、严重程度、否决权)** |
| **判决模式** | 单一模型判决 | **系统与模型双轨判决 + 冲突协调** |
| **推理方向** | 仅前向预测 (Predictive) | **前向预测 + 后向诊断 (Diagnostic)** |
| **数据安全性** | 无权限控制 | **PolicyContext 强制脱敏与过滤** |

**一句话总结**：该架构通过**确定性规则引擎（Critic）**约束 LLM 的自由发挥，利用**结构化事实库（FactStore）**确保证据链可追溯，从而构建起一个可工程化、可校准、可问责的辅助决策系统。
