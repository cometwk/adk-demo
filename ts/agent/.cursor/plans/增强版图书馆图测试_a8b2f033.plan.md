# 增强版图书馆图测试实施计划

## 1. 新增文件清单

新建独立测试目录 `src/v6/demo/tests/graph-2/`，不修改 `1-graph/` 下的任何文件：

- `ontology.ts` — 6 个实体类（Reader、Book、Author、Category、Series、Branch）+ 装饰器 + 10 种关系
- `seed.ts` — 构建含 ~20 实体 + ~30 条边的图数据
- `t1.test.ts` — 10 个测试场景（覆盖 8 条约束 + 图探索 + 聚合）

## 2. 实体与属性设计

### Reader
- `membershipLevel`: 'gold' | 'silver' | 'basic'
- `currentBorrowCount`: number
- `registeredDays`: number
- method: `checkBorrowEligibility(branchMaxBorrow: number)` — 参数需从 Branch 获取

### Book
- `title`, `isbn`, `daysOnShelf`, `totalCopies`, `availableCopies`
- method: `checkAvailability()` — 返回可借状态

### Author
- `name`, `nationality`, `activeBookCount`
- method: `getPopularityScore(avgBorrowThreshold: number)` — 需聚合所有书的借阅量

### Category
- `name`, `isRestricted`: boolean
- `requiredMembershipLevel`: 'gold' | 'silver' | 'basic'

### Series
- `name`, `totalVolumes`: number
- method: `checkReaderProgress(readerBorrowedBookIds: string[])` — 检查是否按顺序阅读

### Branch
- `name`, `maxBorrowPerReader`, `newBookProtectionDays`, `allowInterLibraryLoan`
- method: `findPartnerBranches()` — 返回合作馆列表

## 3. 关系定义（10 种）

| 关系 | 方向 | 说明 |
|------|------|------|
| borrows | Reader → Book | 当前借阅 |
| overdue | Reader → Book | 逾期未还 |
| reserves | Reader → Book | 预约 |
| registered_at | Reader → Branch | 注册所在分馆 |
| written_by | Book → Author | 作者 |
| belongs_to | Book → Category | 类目归属 |
| part_of | Book → Series | 系列归属 |
| available_at | Book → Branch | 哪些分馆有此书 |
| specializes_in | Author → Category | 作者专长类目 |
| partners_with | Branch → Branch | 馆际合作（自引用） |

## 4. 测试场景矩阵（10 个）

| 场景 | 测试目标 | 规则 | 推理跳数 |
|------|---------|------|---------|
| S1 | 全部通过 → 允许借阅 | C1-C4 均不触发 | 2 跳 |
| S2 | 借阅上限触发（需从 Branch 拿 max） | C1 | 2 跳 |
| S3 | 逾期阻断（需遍历 overdue 边确认） | C2 | 1 跳遍历 |
| S4 | 限制类目 — 跨实体会员等级比较 | C4 | 2 跳 + 跨实体 |
| S5 | 新书保护（需从 Branch 拿保护天数） | C3 | 2 跳 |
| S6 | 系列顺序检查 — 多跳集合推理 | C5 | 3-4 跳 |
| S7 | 跨馆调拨 — 链式分馆遍历 | C6 | 3 跳 |
| S8 | 热门作者聚合 — 反向遍历 + 聚合 | C7 | 2 跳 + 聚合 |
| S9 | 预约上限 — 反向计数聚合 | C8 | 反向聚合 |
| S10 | 综合决策 — 多规则叠加（C1+C4+C5） | 多条 | 全路径 |
