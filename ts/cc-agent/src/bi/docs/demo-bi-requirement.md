# BI 分析需求点

> 基于 `docs/demo-bi-ddl.md` 的业务模型，梳理潜在的分析需求

---

## 1. 交易分析

### 1.1 交易趋势分析
- **日/周/月交易量趋势**：按时间维度查看交易额和订单数的变化
- **交易额同比/环比**：对比不同时期的交易增长情况
- **交易高峰分析**：识别交易高峰日/周，辅助运营决策

### 1.2 商户交易排名
- **商户交易额 TOP N**：按交易额排序的商户排行
- **商户订单量 TOP N**：按订单数排序的商户排行
- **大额交易占比分析**：大额订单（如 >1000 元）在总交易中的占比
- **商户交易额分布**：各交易额区间的商户数量分布

### 1.3 通道维度分析
- **各通道交易占比**：不同支付通道的交易额/订单数占比
- **通道-商户交叉分析**：各通道下商户的交易表现
- **通道机构商户活跃度**：各机构商户号的交易频次

### 1.4 商户活跃度分析
- **活跃商户数**：指定时间段内有交易的商户数
- **沉睡商户预警**：超过 N 天无交易的商户列表
- **新增商户交易表现**：新进件商户的首月/首季交易情况
- **商户生命周期**：从进件到首笔交易的时间间隔分析

---

## 2. 分润分析

### 2.1 分润总览
- **日/周/月分润趋势**：按时间维度查看分润收入变化
- **净分润率**：`net_profit / total_trade_amt`，反映实际分润效率
- **退款对分润的影响**：`total_refund_deduct / total_profit`，退款扣除占比

### 2.2 代理商分润排名
- **分润收入 TOP N 代理商**：按分润总收入排序
- **净分润 TOP N 代理商**：按净分润排序
- **自进件 vs 全部分润对比**：`own_net_profit / net_profit`，自进件分润占比

### 2.3 分润结构分析
- **MERCH 代理 vs CHAN 代理分润占比**：按 `agent_type` 维度的分润分布
- **分润费率分布**：各代理商分润费率的分布区间
- **固定模式 vs 占比模式**：两种分润模式的分润额对比

### 2.4 退款分析
- **退款率趋势**：`refund_cnt / order_cnt`，按时间维度的退款率
- **退款金额占比**：`total_refund_amt / total_trade_amt`
- **代理商退款率排名**：按退款率排序的代理商

---

## 3. 代理商分析

### 3.1 代理商层级分析
- **各层级代理商数量**：按代理商在层级树中的绝对层级（`agent_closure` 中从根节点到该节点的 `depth` 值）统计各层级代理商数。注意：`depth` 是祖先-后代对的距离值，需约束 `ancestor_id` 为根节点（`parent_id = 0`）才能得到绝对层级
- **层级分润贡献**：各层级代理商的分润贡献占比
- **团队规模分析**：每个代理商名下的下级代理商数量和商户数量

### 3.2 代理商业绩分析
- **代理商名下商户交易总额**：通过 `agent_rel` 关联计算
- **代理商进件能力**：`apply=1` 的商户数量及交易贡献
- **代理商费率竞争力**：对比同类代理商的分润费率

### 3.3 代理商活跃度
- **活跃代理商数**：指定时间段内有分润记录的代理商
- **代理商发展趋势**：按月新增代理商数量
- **代理商流失预警**：连续 N 天无分润的代理商

---

## 4. 进件分析

### 4.1 进件转化率
- **申请成功率**：`status = SUCCESS 的数量 / 总申请数`
- **进件失败原因分析**：按 `status_reason` 分组的失败原因统计
- **进件时效**：从申请到成功的平均耗时（**数据限制**：`apply` 表无状态变更时间戳，`updated_at` 会在任何字段修改时更新，仅可作近似参考）

### 4.2 进件趋势
- **日/周/月进件量趋势**：按时间维度的进件申请量
- **各代理商进件量排名**：按进件数排序的代理商
- **进件质量**：进件成功商户的后续交易表现

### 4.3 通知分析
- **通知成功率**：`notify = 2 (SUCCESS) 的数量 / status >= 2 的申请数`（仅统计进入通知阶段的记录）
- **通知重试分析**：平均重试次数，重试后成功率

---

## 5. 费率分析

### 5.1 费率结构分析
- **商户费率分布**：各费率区间的商户数量
- **平均费率趋势**：按时间维度的平均商户费率变化
- **费率与交易额相关性**：高/低费率商户的交易表现对比

### 5.2 利润空间分析
- **平台利润率**：`商户费率 - 通道费率 - 代理费率` 的空间分析
- **分润成本占比**：代理分润占商户费率的比例
- **利润空间趋势**：按时间维度的利润空间变化

---

## 6. 综合交叉分析

### 6.1 商户-代理商-交易三维分析
- **各代理商名下商户的交易表现**：按代理商维度查看商户交易
- **进件来源交易贡献**：不同代理商进件的商户对总交易额的贡献
- **代理商-通道交叉**：各代理商在不同通道的业务量

### 6.2 时间序列交叉
- **新商户 vs 老商户交易对比**：按进件时间分组的交易表现
- **季节性分析**：识别交易/分润的季节性波动
- **节假日效应**：节假日期间的交易变化

### 6.3 风险预警
- **交易异常检测**：商户交易额突增/突降预警
- **退款异常**：退款率异常升高的商户/代理商
- **费率异常**：费率偏离正常区间的商户/代理关系

---

## 7. 运营看板需求

### 7.1 管理层看板
- 日/月交易总额与趋势
- 日/月分润总额与趋势
- 活跃商户数、活跃代理商数
- 进件转化率

### 7.2 代理商看板
- 自己名下商户交易概况
- 分润收入与趋势
- 自进件商户 vs 全部商户分润对比
- 名下商户活跃度

### 7.3 运营看板
- 进件审批队列
- 沉睡商户预警列表
- 退款率异常预警
- 代理商发展健康度
