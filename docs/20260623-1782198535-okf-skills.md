# Open Knowledge Format (OKF) Skills

**将你的数据已有的结构——数据库模式、列注释、外键、文件树、提交历史——转化为可浏览的、Agent 可读的知识目录。**

OKF Skills 是小型的、确定性的连接器，可以从任何数据源**生成** [Open Knowledge Format](https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf) 包（Markdown + YAML 目录），让 LLM 用基于事实的描述来**富化**它，将其**可视化**为交互式图谱，并将描述**同步**回数据源。提取是纯函数式且可复现的——没有嵌入模型——所以循环中唯一的 LLM 是*你的* Agent，由指令引导。每个连接器都是一个独立的可移植二进制文件，通过 MCP 自描述，因此可以直接插入任何 Agent 框架。

## 🚀 快速开始

### 1. 安装

最快的方式——通过 [skills.sh](https://www.skills.sh) 注册中心将整套技能添加到你的 Agent：

```bash
npx skills add xSAVIKx/okf-skills
```

…或者从克隆构建 + 安装二进制文件（所有连接器 + `okf-mcp` 服务器）：

```bash
git clone https://github.com/xSAVIKx/okf-skills && cd okf-skills
./install.sh                       # 构建并安装到 ~/.local/bin
# ./install.sh /usr/local/bin      # …或你选择的目录
```

…或者只获取你需要的，无需克隆（需要 Go 1.24+）：

```bash
go install github.com/xSAVIKx/okf-skills/okf-mcp@latest            # MCP 服务器
go install github.com/xSAVIKx/okf-skills/skills/okf-sqlite@latest  # 单个连接器
```

确保安装目录在你的 `PATH` 上——`okf-sqlite --version` 应该能打印版本号。

### 2. 连接到你的 Agent

`okf-mcp` 发现已安装的 `okf-*` 技能，并将每个命令暴露为名为 `<skill>__<command>` 的 MCP 工具（例如 `okf-sqlite__produce`、`okf-viz__render`）。只需一次配置：

**Claude Code**
```bash
claude mcp add okf-skills okf-mcp
```

**Gemini CLI**
```bash
gemini mcp add okf-skills okf-mcp
```

**Cursor / Codex / Windsurf / 任何 MCP 客户端** — 添加 MCP 服务器条目：
```json
{
  "mcpServers": {
    "okf-skills": { "command": "okf-mcp" }
  }
}
```

`okf-mcp` 默认扫描你的 `PATH` 中的技能；使用 `"args": ["--skills-dir", "/path/to/bin"]` 指定目录。（Codex 使用 TOML 等价写法：`[mcp_servers.okf-skills]` 加 `command = "okf-mcp"`。）

### 3. 使用

用自然语言向你的 Agent 提问——它会为你驱动工具：

> *"对我的 SQLite 数据库 ./app.db 建立目录，富化表描述，并渲染可视化图谱。"*

它会调用 `okf-sqlite__produce`（→ 生成 OKF 包），按照 `okf-enrich` 指导编写基于事实的描述，然后 `okf-viz__render` 生成一个独立的 `index.html`。喜欢命令行？手动执行相同流程：

```bash
okf-sqlite produce --db ./app.db --out ./catalog --profile   # 提取 → OKF 包
# 富化描述（手动，或由遵循 okf-enrich 指导的 Agent 完成）
okf-sqlite ingest  --db ./app.db --bundle ./catalog --sync    # 将描述推送回去
okf-viz    render  --bundle ./catalog                         # → ./catalog/index.html
```

## 为什么选择 OKF？

你的数据源充满了从未到达需要它的人——或 Agent——那里的结构：模式、`COMMENT ON` 文本、外键、索引、仓库的文件树和提交历史。OKF 将其捕获为纯 Markdown + YAML——**可 diff、可 grep、可在 PR 中审查**，人类、LLM 或 `grep` 同样可读。连接器**确定性地**提取（没有变更时重新运行是无操作），你的 Agent 在此基础上添加含义，目录可以往返同步回数据源。没有专有存储，没有锁定——只有文件。

本文档的其余部分是参考：仓库布局、每个技能的详细说明、测试和发布。大多数技能是自包含的 Go 二进制文件；Agent 指导技能（`okf-enrich`、`okf-reader`、`okf-producer-generator`）仅包含指令（`SKILL.md`，无二进制文件）。

## 1. 独立技能（`skills/`）

`skills/` 下的每个文件夹是一个自包含的 Go 模块，包含：
- `SKILL.md`：指导任何编码 Agent（如 Claude Code、Cursor、Copilot）如何执行连接器。
- Go 源代码（`main.go` 和 `go.mod`）：编译为单个可移植二进制文件。

每个技能通过 `schema` 子命令自描述，输出描述其命令、标志和参数的 JSON。

### 可用连接器

| 技能 | 数据源 | 关键特性 |
|---|---|---|
| `okf-sqlite` | SQLite 数据库 | 通过 `modernc.org/sqlite` 无 CGO |
| `okf-mysql` | MySQL 数据库 | 基于 DDL 的注释同步 |
| `okf-postgresql` | PostgreSQL 数据库 | 基于 `COMMENT ON` 的同步 |
| `okf-bigquery` | Google Cloud BigQuery | GCP 凭据 / API 密钥 |
| `okf-fs` | 本地文件系统 | `.okfignore` 和 `.okf-metadata.yaml` 支持 |
| `okf-git` | Git 仓库 | 提交历史和文件级元数据 |

### 命令

所有连接器支持三个子命令：
- **`produce`**：从数据源提取元数据并生成 OKF 包。四个 SQL 连接器（`okf-sqlite`、`okf-mysql`、`okf-postgresql`、`okf-bigquery`）还支持 `produce` 上的 `--sample` 和 `--profile` 标志。
- **`ingest`**：读取 OKF 包并将描述比较/同步回数据源。
- **`schema`**：输出技能命令、标志和参数的 JSON 描述。

### 如何安装或构建技能

每个技能都是已发布的 Go 模块，所以最简单的安装不需要克隆（需要 Go 1.24+）：
```bash
go install github.com/xSAVIKx/okf-skills/skills/okf-sqlite@latest   # → $(go env GOPATH)/bin/okf-sqlite
```
或者从克隆构建——导航到技能目录并运行：
```bash
go build ./...
```

## 2. 富化指导技能（`okf-enrich`）

位于 `skills/okf-enrich/`，这是一个仅包含指令的技能（`SKILL.md`）——没有二进制文件，没有嵌入模型。它教导 Agent 自己的 LLM 如何富化 OKF 包：找到描述薄弱或缺失的概念，基于模式加上 `Data Profile`/`Sample` 章节编写有依据的新描述，将它们写回概念 frontmatter，并（可选地）使用匹配连接器的 `ingest --sync` 推送到数据源。

富化是一个判断任务，所以它作为已在循环中的 LLM 的指导存在，而不是嵌入第二个 LLM 的工具。

---

## 3. MCP 服务器（`okf-mcp`）

`okf-mcp` 是一个通用 MCP（Model Context Protocol）服务器，它发现所有已安装的 `okf-*` 技能二进制文件，并将它们的命令作为 MCP 工具暴露给任何支持 MCP 的框架（Claude Code、Gemini CLI 等）。

不需要定制 Agent。将任何支持 MCP 的框架指向 `okf-mcp`：

```bash
okf-mcp                                    # 发现 PATH 上的技能
okf-mcp --skills-dir /path/to/skills       # 指定技能目录
```

注册为 MCP 服务器后，每个连接器命令（`produce`/`ingest`）都显示为可调用的工具。指导技能（`okf-enrich`、`okf-reader`）不是工具——它们是 Agent 直接加载的 `SKILL.md` 指令。

---

## 4. 摄取指导技能（`okf-reader`）

位于 `skills/okf-reader/`，这是一个仅包含指令的技能（`SKILL.md`）。它教导 AI Agent 如何高效地阅读和导航 OKF 包，最小化上下文 token 开销，防止缓慢的递归目录读取。

---

## 5. 共享库（`okf-go/`）

`okf-go` 模块提供所有技能使用的共享 Go 类型和辅助函数：
- `Frontmatter` / `ConceptDoc` 结构体用于 OKF YAML+Markdown 序列化
- `ReadConceptDoc` / `WriteConceptDoc` 用于解析和写入 OKF 概念文件
- `UpsertSection` / `GetSection` 用于添加或替换 Markdown 正文章节（例如 `Data Profile`、`Sample`）而不破坏周围内容
- `ColumnProfile` 及 `RenderProfileSection` / `RenderSampleSection` 用于 `--profile` / `--sample` 输出
- `SkillSchema` / `PrintSchema` 用于每个技能发出的自描述 `schema` 子命令
- `IgnoreMatcher` 用于 `.okfignore` 通配符支持
- `ReadFolderMetadata` / `WriteFolderMetadata` 用于 `.okf-metadata.yaml`

所有技能以发布版本导入此模块（`github.com/xSAVIKx/okf-skills/okf-go v0.1.0`）。本地开发通过 `go.work` workspace 从磁盘解析，因此对 `okf-go` 的修改无需重新发布即可生效——不需要每个模块的 `replace` 指令。

## 6. 安装技能

直接从发布模块安装任何单个技能（或 `okf-mcp` 服务器）——无需克隆（需要 Go 1.24+）：

```bash
go install github.com/xSAVIKx/okf-skills/skills/okf-sqlite@latest
go install github.com/xSAVIKx/okf-skills/okf-mcp@latest
```

二进制文件安装在 `$(go env GOPATH)/bin`（确保它在你的 `PATH` 上）。

要从克隆一次构建和安装**所有**技能，使用 `install.sh`（或 `make install`）：

```bash
# 安装到 $HOME/.local/bin（默认）
./install.sh

# 安装到自定义目录
./install.sh /usr/local/bin

# 或通过 make
make install
```

安装后，确保目录在你的 `PATH` 上。然后可以直接调用技能（`okf-sqlite produce ...`）或运行 `okf-mcp` 将所有技能暴露为你的编码 Agent 的 MCP 工具。

## 7. 本地测试环境

集成测试位于 `tests/`，覆盖所有连接器、连接器的 `schema` 自描述合约，以及 `okf-mcp` 对已构建技能的发现。提供了 `docker-compose.yml` 来启动预加载模拟数据库的 MySQL 和 PostgreSQL 实例：

```bash
# 1. 就地构建技能二进制文件（集成测试需要，
#    它们在 skills/<name>/<name> 定位二进制文件）。使用 GNU make (Linux/macOS/CI)：
make build
#    没有 make（例如 Windows 上的 Git Bash），分别构建每个连接器和 okf-mcp：
#    for each: cd skills/okf-sqlite && go build -o okf-sqlite .   (and cd okf-mcp && go build -o okf-mcp .)

# 2. （可选）启动 MySQL 和 PostgreSQL 用于数据库集成测试：
cd tests && docker-compose up -d && cd ..

# 3. 运行集成测试套件：
cd tests && go test -v .
```

SQLite、文件系统、Git、`schema` 合约检查和 `okf-mcp` 发现无需 Docker 即可运行；只有 MySQL 和 PostgreSQL 测试用例需要容器（否则会跳过）。

## 8. 规范合规与注册中心发现

每个 `SKILL.md` 遵循 [Agent Skills 规范](https://agentskills.io/specification)：YAML frontmatter 仅暴露规范的顶级键（`name`、`description`、`license`、`compatibility`、`metadata`、`allowed-tools`），项目特定字段（`version`、`author`、`tags`）嵌套在 `metadata` 下。每个 `name` 匹配其技能目录，每个技能声明 `license: Apache-2.0`（仓库采用 Apache-2.0 许可）。使用参考工具验证：`skills-ref validate skills/okf-sqlite`。

根目录的 `skills.sh.json` 清单为 [skills.sh](https://www.skills.sh) 注册中心分组技能，以便在仓库被索引时有组织：

| 组 | 技能 |
|---|---|
| Database Connectors | `okf-sqlite`、`okf-mysql`、`okf-postgresql`、`okf-bigquery` |
| Filesystem & Git | `okf-fs`、`okf-git` |
| Agent Guidance | `okf-reader`、`okf-enrich`、`okf-producer-generator` |
| Visualization | `okf-viz` |

`okf-mcp` 有意从注册中心清单中省略：它是通过 MCP 暴露技能的主机服务器（位于 `skills/` 之外），本身不是可发现的技能。

## 9. Producer 生成器技能（`okf-producer-generator`）

位于 `skills/okf-producer-generator/`，这是一个仅包含指令的技能（`SKILL.md`）——没有二进制文件。它是项目中"编写 Producer"的入门指南：它附带了 [官方 OKF 规范](https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf)的快照（`okf-SPEC.md`）、`okf-go` 库 API 参考（`okf-go-api.md`），以及编写与现有六个连接器匹配的新连接器的分步指南，而非逆向工程。

它涵盖了架构原则（**无嵌入 LLM** 的确定性提取、`okf-go` 作为 OKF 类型的单一来源、`schema` 作为 MCP 发现合约）、`produce`/`ingest`/`schema` 命令接口、密钥处理和 `--sync` 约定，以及完整的注册检查清单（`go.work`、`Makefile`、`install.sh`、`skills.sh.json`、文档和测试）。

当扩展项目到尚未覆盖的数据源时加载它——例如 MongoDB、Redis、Kafka、CSV 目录或 HTTP API。

## 10. 可视化技能（`okf-viz`）

位于 `skills/okf-viz/`，这是一个 Go CLI 消费者技能，将任何 OKF 包渲染为单个独立的交互式 `index.html`，写在 `index.md` 旁边。它生成一个三栏浏览器：导航器（树 + 类型/标签过滤器 + 全文搜索）、带有七种可切换布局和可折叠边类型图例的 Cytoscape 图谱，以及渲染的概念阅读器。

```bash
# 构建
cd skills/okf-viz && go build -o okf-viz .

# 渲染包（CDN 模式——需要网络获取图谱库）
./okf-viz render --bundle path/to/bundle

# 完全离线渲染——所有图谱 JS 内联，无需网络
./okf-viz render --bundle path/to/bundle --offline

# 额外选项
./okf-viz render --bundle path/to/bundle --theme dark --title "My Catalog"
```

`render` 命令默认写入 `<bundle>/index.html`（使用 `--out` 覆盖）。输出是单个可移植的 HTML 文件，无需服务器——在任何浏览器中打开。图谱显示目录层次结构的包含边（虚线、灰色）和概念之间显式 `[text](../concept.md)` 引用的实线交叉链接边（彩色）。

## 持续集成与发布

CI（[`.github/workflows/ci.yml`](.github/workflows/ci.yml)）在每个 PR 上运行：
gofmt、`go vet`、每个模块的构建、单元测试，以及 Docker 支持的
集成测试套件（SQLite/FS/Git 始终运行；MySQL/PostgreSQL 通过服务容器）。

发布从 [Conventional Commits](https://www.conventionalcommits.org/)
使用 [release-please](https://github.com/googleapis/release-please) 自动化：将工作合并
到 `master` 会维护一个发布 PR，合并该 PR 会以 Go 的 `<path>/vX.Y.Z` 格式
标记每个变更的模块、创建 GitHub Release，并预热模块代理。
详见 **[RELEASING.md](RELEASING.md)** 了解完整流程和一次性仓库设置。
