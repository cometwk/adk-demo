## 线上下单线下支付下单

### 🏪 场景设定：建材批发市场（“宏达建材城”）

- **市场方**：宏达建材城（使用建行“惠市宝”平台）
- **商户**：A01号摊位“王老板瓷砖店”
- **消费者**：李先生，要买一批瓷砖用于装修
- **支付方式**：李先生用**建行信用卡**在线下POS机支付

---

## 🔁 三步流程详解（带数据示例）

### 第一步：市场平台调用「生成支付订单接口」

> ✅ 目的：在惠市宝系统中“占位”并绑定分账规则

**触发时机**：李先生在宏达建材城的微信小程序上下单：
- 商品：800×800瓷砖 × 50片
- 单价：¥60/片
- 总金额：¥3,000
- 指定摊位：A01（王老板）

**市场系统后台自动调用惠市宝API**：
```http
POST /api/v1/createOrder
{
  "orderNo": "HD202604211001",   // 市场自定义订单号
  "amount": 3000,
  "merchantId": "A01",
  "payType": "offline_pos",
  "splitRule": {
    "marketFeeRate": 0.03,       // 市场管理方收3%（¥90）
    "vendorAmount": 2910         // 摊主实收¥2910
  }
}
```

✅ **惠市宝返回**：`{"code": 200, "msg": "订单创建成功"}`  
→ 此时惠市宝内部记录了一笔 **“待支付”订单 HD202604211001**

---

### 第二步：线下完成 POS 支付

> ✅ 目的：完成真实资金交易

**操作过程**：
1. 李先生到建材城A01摊位提货；
2. 告知王老板：“我在小程序下单了，订单号 HD202604211001”；
3. 王老板打开**建行POS机**（已接入惠市宝），选择“消费”；
4. 输入金额：**3000.00 元**（必须与订单金额一致）；
5. 李先生刷**建行信用卡**，输入密码；
6. POS机打印小票，交易成功，流水号为：`POS2026042111250088`。

⚠️ 注意：此时**资金已进入建行清算系统**，但惠市宝**还不知道这笔支付和订单 HD202604211001 有关联**！

---

### 第三步：市场方调用「更新线下支付订单状态接口」

> ✅ 目的：打通“支付行为”与“订单系统”，触发分账

**触发时机**：王老板在市场管理APP点击“确认收款”，或系统自动轮询POS流水后匹配。

**市场系统调用惠市宝API**：
```http
POST /api/v1/updateOfflinePayStatus
{
  "orderNo": "HD202604211001",      // 关键！关联第一步的订单
  "payStatus": "SUCCESS",
  "payTime": "2026-04-21T11:25:30",
  "posTraceNo": "POS2026042111250088", // POS流水号（用于对账）
  "actualAmount": 3000
}
```

✅ **惠市宝收到后**：
1. 校验 `orderNo` 是否存在且金额匹配；
2. 将订单状态从“待支付”改为“已支付”；
3. **立即执行分账**：
   - ¥90 划入“宏达建材城”对公账户（管理费）
   - ¥2910 划入“王老板”绑定的收款账户
4. 向市场平台、王老板、李先生推送“支付成功+分账完成”通知。

---

## 🔗 三步如何逻辑自洽？

| 步骤 | 解决什么问题 | 依赖前一步吗？ | 数据如何衔接？ |
|------|-------------|---------------|----------------|
| 1. 生成订单 | 预留订单 + 定义分账规则 | —— | 生成唯一 `orderNo` |
| 2. POS支付 | 完成真实资金收付 | 否（但需人工/系统确保金额=订单金额） | 产生 `posTraceNo` |
| 3. 更新状态 | 告知惠市宝“这笔POS支付对应哪个订单” | **是！必须有 step1 的 orderNo** | 用 `orderNo` + `posTraceNo` 关联 |

> 💡 **核心逻辑闭环**：  
> **订单号（orderNo）是贯穿三步的“唯一纽带”**。  
> 没有它，惠市宝无法知道哪笔POS交易对应哪个线上订单，也就无法自动分账。

---

## ❌ 如果缺少某一步，会发生什么？

- **没做第1步**：POS支付成功，但惠市宝无此订单 → 资金无法分账，只能手工处理。
- **没做第3步**：惠市宝始终认为订单未支付 → 摊主收不到钱，可能误以为客户逃单。
- **第2步金额 ≠ 第1步**：第3步调用会被惠市宝拒绝（防欺诈/错付）。

---

### ✅ 总结

这三步构成一个**“线上预约 + 线下履约 + 系统闭环”** 的完整O2O资金流：
> **生成订单（计划） → POS支付（执行） → 更新状态（确认）**

既满足了消费者线上选品、线下体验的需求，又让市场方实现了**自动化分账、合规清结算、精准对账**——这正是惠市宝的核心价值。
